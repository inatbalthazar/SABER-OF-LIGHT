# Saber of Light - Stand-Up & Demo Presentation

## 1. บทพูดสำหรับ Stand-Up (อัปเดตงานประจำวัน)

**"สวัสดีครับ/ค่ะ อัปเดตงานในส่วนของโปรเจกต์ Saber of Light Store นะครับ/คะ"**
- **สิ่งที่ทำเสร็จแล้ว (What I did):** ตอนนี้ได้ทำการสร้างหน้า Store ของร้านขายดาบเลเซอร์เสร็จเรียบร้อยแล้ว โดยมีการปรับธีมให้เป็น Dark Mode เพื่อให้เข้ากับสไตล์ Sci-Fi และได้ทำการ Refactor (ปรับปรุง) โค้ด HTML และ CSS ครั้งใหญ่ เพื่อลดการเขียนคลาสซ้ำซ้อน ทำให้โค้ดอ่านง่ายและกระชับขึ้นมากครับ
- **กำลังทำอะไรอยู่ (What I'm doing):** ตอนนี้กำลังนำเสนอ Demo ของหน้าเว็บ และเตรียมส่งมอบงานครับ
- **ปัญหาที่พบ (Blockers):** ในช่วงแรกมีปัญหาเรื่องการจัด Layout และคลาสใน CSS ที่ซ้ำซ้อนกันเยอะมาก แต่ตอนนี้แก้ไขด้วยการใช้ Flexbox และ Tag Selectors แทนเรียบร้อยแล้ว ไม่มี Blocker ครับ

## 2. บทพูดสำหรับ Demo (นำเสนอหน้าเว็บ)

**"เดี๋ยวเรามาดูหน้าเว็บของจริงกันเลยนะครับ"**
*(เปิดหน้าเว็บโชว์)*
1. **Hero Section:** เริ่มจากด้านบนสุด เรามี Topbar เล็กๆ สำหรับแจ้งโปรโมชัน ถัดมาคือ Hero Section ที่ใช้ภาพพื้นหลังแบบเต็มตา พร้อมตัวหนังสือที่ใช้เอฟเฟกต์เรืองแสง (Text-shadow) เพื่อดึงดูดสายตา
2. **Categories:** พอลากลงมาจะเจอหมวดหมู่สินค้า ผมออกแบบให้เป็นแนวการ์ด เมื่อเอาเมาส์ไปชี้ (Hover) จะเห็นว่ากรอบมีการเปลี่ยนสีและเรืองแสง พร้อมกับรูปภาพที่ซูมเข้าเล็กน้อย ทำให้เว็บดูมีมิติและน่าคลิกครับ
3. **Products & Flexbox Layout:** ถัดมาคือส่วนของสินค้า (Sabers, Display Dock, Accessories) จะเห็นว่าสินค้าเรียงกันอย่างเป็นระเบียบ ซึ่งตรงนี้ผมได้ใช้เทคนิคการจัดวางแบบตารางที่ปรับเปลี่ยนได้ และถ้าเราลองย่อหน้าจอเว็บดู *(แสดงการย่อหน้าจอ)* สินค้าจะถูกปัดตกลงมาเรียงใหม่ให้พอดีกับหน้าจอมือถือและแท็บเล็ตโดยอัตโนมัติครับ
4. **Consistency:** นอกจากนี้ รูปสินค้าทุกชิ้นในร้าน ผมได้ตั้งค่าให้มีเอฟเฟกต์ Hover เหมือนกันทั้งหมด เพื่อให้ประสบการณ์ผู้ใช้ (UX) เป็นไปในทิศทางเดียวกันครับ

## 3. คำอธิบายการเขียนโค้ด (โชว์ความเข้าใจ HTML & CSS)

**"ในส่วนของเบื้องหลังการเขียนโค้ด ผมได้นำความรู้ที่เรียนมาประยุกต์ใช้ดังนี้ครับ:"**

### 1. Semantic HTML (โครงสร้างที่สื่อความหมาย)
แทนที่จะใช้ `<div>` ครอบทุกอย่าง ผมเลือกใช้ Tag ที่สื่อความหมายชัดเจน เช่น:
- `<header>` สำหรับส่วนบนสุด
- `<main>` สำหรับเนื้อหาหลัก
- `<section>` แบ่งหมวดหมู่ต่างๆ
- `<footer>` สำหรับส่วนท้าย
ช่วยให้โค้ดเป็นระเบียบ และดีต่อ SEO ครับ

### 2. CSS Flexbox (การจัด Layout)
การเรียงสินค้าในร้าน ผมไม่ได้ใช้การกะระยะด้วย margin ธรรมดา แต่ใช้ **Flexbox** ผ่านคลาส `.grid`:
```css
.grid {
    display: flex;
    flex-wrap: wrap;
    justify-content: center;
    gap: 30px;
}
```
คำสั่ง `flex-wrap: wrap` ช่วยให้เมื่อพื้นที่ไม่พอ สินค้าจะไหลลงมาบรรทัดใหม่เอง และใช้ `gap` ในการเว้นช่องไฟให้เท่ากันอย่างสวยงาม

### 3. Responsive Design (การรองรับมือถือ)
ผมใช้ `@media queries` ในการจัดการหน้าจอหลายขนาด:
```css
@media (max-width: 768px) {
    .grid > .item {
        width: calc(50% - 15px);
    }
}
@media (max-width: 480px) {
    .grid > .item, .dock {
        flex: 0 0 100%;
        width: 100%;
    }
}
```
ทำให้สินค้าที่แสดง 3-4 ชิ้นบนคอมพิวเตอร์ จะเหลือ 2 ชิ้นบนแท็บเล็ต และ 1 ชิ้นเต็มจอเมื่อเปิดบนมือถือ

### 4. CSS Selectors & Code Refactoring (การลดความซ้ำซ้อนของโค้ด)
ผมให้ความสำคัญกับ "Clean Code" โดยเปลี่ยนจากการใส่คลาสให้ทุกๆ แท็ก (เช่น `<h2 class="sec-title">`, `<button class="btn-buy">`) ไปใช้ **Descendant Selectors** (การอ้างอิงลำดับชั้น) แทน:
- ใช้ `section h2 { ... }` เพื่อคุมหัวข้อหลักของทุก Section
- ใช้ `.item > button { ... }` เพื่อจัดการปุ่มสั่งซื้อในตัวสินค้า
การทำแบบนี้ทำให้ไฟล์ HTML สะอาดขึ้นมาก และ CSS ก็สั้นลง ลดการใช้คลาสที่ทำหน้าที่เหมือนกันไปได้นับสิบคลาสครับ

### 5. Pseudo-classes & Transitions (ลูกเล่นแอนิเมชัน)
เพื่อให้เว็บดูน่าสนใจ ผมใช้ `:hover` ควบคู่กับ `transition`:
```css
.item img, .dock img {
    border: 1px solid #444;
    transition: transform 0.4s ease, border-color 0.3s, box-shadow 0.3s;
}
.item:hover img, .dock:hover img {
    transform: scale(1.05);
    border-color: #1fa0d9;
    box-shadow: 0 0 15px rgba(31, 160, 217, 0.4);
}
```
คำสั่งเหล่านี้คือเบื้องหลังของ "การสว่างขึ้น" และ "ซูมเข้า" ที่ทำให้ผู้ใช้รู้ว่าเขากำลังชี้เมาส์ที่สินค้าตัวไหนอยู่ครับ
